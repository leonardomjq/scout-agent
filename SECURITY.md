# Security Policy

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, send an email to **leonardo@pdt.digital** with:

- A description of the vulnerability
- Steps to reproduce (or a proof-of-concept)
- The potential impact

### What to expect

- **Acknowledgement** within 48 hours
- A plan for a fix or mitigation within 7 days
- Credit in the release notes (unless you prefer anonymity)

## Supported versions

| Version | Supported |
| ------- | --------- |
| 2.x     | Yes       |
| 1.x     | No        |

## Scope

The following are in scope for Scout Daily:

- XSS or injection via the `/api/subscribe` endpoint
- Data exposure through API routes or static assets
- Server-side injection vulnerabilities
- Supply chain issues (dependency vulnerabilities)

Thank you for helping keep Scout Daily and its users safe.
